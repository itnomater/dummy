Library FileSystem

Helper functions to manage the files and directories.

It uses a standard shell commands like: stat, chmod or build-in test, but wrapped them in the one interface.

In short it can do:

  • Check file type (regular, directory, symlink, etc.).
  • Get a file permissions, modification date or size.
  • Generate MD5 or SHA1 hash of the file.
  • Change a file permissions.
  • Unlink a file.

Check a node type or state:

  • fs_is_exist - Is the file exist?
  • fs_is_readable - Is the file readable?
  • fs_is_writable - Is the file writable?
  • fs_is_executable - Is the file executable?
  • fs_is_dir - Is the path a directory?
  • fs_is_symlink - Is the path a symlink?
  • fs_is_reg - Is the path a regular file?
  • fs_is_block - Is the path a block device?
  • fs_is_character - Is the path a character device?
  • fs_is_pipe - Is the path a pipe?
  • fs_is_socket - Is the path a socket?
  • fs_is_empty - Is the file/directory empty?
  • fs_is_salted - Is the file encrypted using openssl with salt?
  • fs_is_compressed - Is the file compressed using xz command?
  • fs_mdate - Get the file modification date.
  • fs_mtime - Get the file modification time.
  • fs_mdatetime - Get the file modification full date (date + time).
  • fs_mtimestamp - Get the file modification UNIX timestamp.
  • fs_perm - Get the file permissions (in octal).
  • fs_size - Get the file size (in bytes).
  • fs_is_support_permissions - Does the file system support permissions?

Make node changes:

  • fs_chmod - Set the file permissions.
  • fs_set_mtimestamp - Set the file modification time.
  • fs_mksym - Make a symbolic link to the file.
  • fs_mkdir - Make the directory (if doesn’t exist only).
  • fs_rmdir - Remove the directory (if is empty only).
  • fs_unlink - Remove the file.

Generate checksums from the file:

  • fs_md5 - Generate the MD5 hash of the file.
  • fs_sha1 - Generate the SHA1 hash of the file.
